# Running Phoenix OS on the official v86 website

Phoenix OS builds real v86-usable images. You do **not** need to host your own emulator page.
Use the official v86 site at <https://copy.sh/v86/> and select either the generated hard-disk image or ISO.

## Recommended: persistent hard-disk image

Use this when you want Phoenix to act like a normal installed desktop OS with a writable hard drive. This is also the best option for Wine, because `~/.wine` and installed Windows files persist across boots:

```bash
./scripts/build-v86-disk.sh
```

The script builds the `v86-disk` KIWI profile and copies the final raw disk image to:

```text
dist/phoenix-v86.img
```

Boot it on <https://copy.sh/v86/>:

1. Open <https://copy.sh/v86/>.
2. Go to **Setup**.
3. In **Hard disk image**, choose `dist/phoenix-v86.img` from your computer.
4. Set **Memory size** to at least `512` MB for the JWM-style desktop.
5. Set **Boot order** to `Hard Disk / CD / Floppy`.
6. Click **Start Emulation**.

## Live ISO image

Use this when you specifically want a CD-style live ISO:

```bash
./scripts/build-v86-iso.sh
```

The script builds the `v86-live` KIWI profile and copies the final ISO to:

```text
dist/phoenix-v86.iso
```

Boot it on <https://copy.sh/v86/>:

1. Open <https://copy.sh/v86/>.
2. Go to **Setup**.
3. In **CD image**, choose `dist/phoenix-v86.iso` from your computer.
4. Set **Memory size** to at least `512` MB for the desktop.
5. Set **Boot order** to `CD / Hard Disk / Floppy`.
6. Click **Start Emulation**.

## Notes

- The image file stays local in your browser when selected on the v86 page.
- `v86-disk` and `v86-live` are still `i586` and BIOS bootable.
- The kernel command line keeps `console=ttyS0`, `console=tty0`, and `nomodeset` so boot output is visible and graphics stay simple in v86.
- Browser networking in v86 depends on the network proxy setting on the v86 page.
